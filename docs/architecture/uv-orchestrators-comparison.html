UV Orchestrators

A side-by-side comparison of the current legacy DAG forest backend and the new hedge-poset backend. Both are selected through UVgenerationSettings::orchestrator.

Default

legacy_dag_forest remains the reference path.

New Backend

hedge_poset exposes trace levels and compute-store reuse; disconnected policies are still being validated.

Validation

compare runs both backends, checks normalized results, and returns legacy output.

Shared Operational Methods

Highlighted method names are the intended common lifecycle. The backends keep different internals, but the orchestration reads as the same set of operations wherever possible.

unfold

Turn the wood into an executable forest representation.

compute

Populate local terms, integrated state where supported, and final terms.

orientation_parametric_exprs

Assemble cut-indexed parametric integrands from computed terms.

pole_part_of_ends

Extract the integrated renormalization output from terminal terms.

final_integrand_builder

Reuse final-integrand assembly through the forest-node invariant.

Partial Reuse Local 3D

The Local3DApproximation::kernel implementation is shared. The outer local algorithm is not shared: legacy walks one dependent DAG node at a time, while hedge-poset walks trace levels from a cut root. The local kernel approximates a full incoming atom, so disconnected local 3D is not solved by multiplying full local_step outputs; that duplicates the root/common ancestor. The unresolved hedge-poset policy is to make disconnected local composition operational: return to the common ancestor for each component and multiply only the operation effects. 

// legacy Approximation::compute
sum_3d += Local3DApproximation {}
    .kernel(&ctx, self, dependent, &local)?;
sum_3d -= Local3DApproximation {}
    .kernel(&ctx, self, dependent, &integ)?;
// hedge-poset OperationNode::local, current structural shape
for level in levels.iter_levels_top_down() {
    let mut level_product: Option<ResidueSelectedTerms> = None;
    let dependent_integrated_finite =
        ctx.dependent_integrated_finite(&completed_levels)?;

    for op in level.iter_leaf_ops() {
        let (current, given) =
            ctx.wood.current_given_pair(op.data, order);
        let stepped = ctx.local_step(
            &acc,
            &current,
            &given,
            &local,
            dependent_integrated_finite.as_ref(),
        )?;

        // This multiplication is structural, not yet the final
        // disconnected-local policy.
        if let Some(product) = &mut level_product {
            product.multiply_assign_aligned(
                stepped,
                "hedge-poset level product",
            )?;
        } else {
            level_product = Some(stepped);
        }
    }
}

The reusable boundary is the analytic kernel: it only needs ForestNodeLike inputs. Scheduling, caching, and disconnected-component composition belong to the orchestrator.

Reused Final Integrand

Final-integrand assembly is shared directly through FinalIntegrandNode. Legacy calls it on Approximation; hedge-poset converts an operation into an OwnedForestNode. Both types implement ForestNodeLike, so the builder owns the numerator, orientation localization, finite integrated subtraction, and color simplification path.

Legacy execution is a topological DAG computation over Approximation nodes.

    match self.dag.nodes[n].parents.len() {

Traversal

Hedge-poset traversal normalizes each compatible cut order so root/frontier state is present.

fn compatible_topological_order(
    &self,
    subset: &SuBitGraph,
) -> Result<Vec<NodeIndex>> {
    let mut order = self.graph.topo_sort_kahn_of(subset)?;
    if !order.contains(&self.root) {
        order.insert(0, self.root);
    }
    Ok(order)
}

Independent Components

Hedge-poset exposes independent operations as trace levels. Integrated disconnected terms are the simpler case: component integrated CTs can be multiplied as scalar terms. Local 3D is different because local_step applies an operation to a full incoming accumulator; disconnected local composition must return to the common ancestor for each component and multiply only operation effects. 

for level in levels.iter_levels_top_down() {
    let mut mul = Atom::one();

    for op in level.iter_leaf_ops() {
        let (current, given) = wood.current_given_pair(op.data, order);
        mul *= -integrated_orchestrator.kernel(
            &uvctx,
            &current,
            &given,
            &acc,
        )?;
    }

    acc = mul;
}
// Current local 3D level multiplication is structural only.
// It is not the final disconnected-local semantics.
product.multiply_assign_aligned(
    stepped,
    "hedge-poset level product",
)?;

Output Assembly

Legacy asks each computed DAG forest for its orientation-parametric expression.

for (forest, cuts) in
    self.forests.iter().zip(self.cuts.cuts)
{
    let mut integrands =
        forest.orientation_parametric_expr(
            graph,
            settings.add_sigma,
        )?;
    if !settings.keep_sigma {
        integrands.values_mut().for_each(|s| {
            *s = s.replace(function!(GS.if_sigma, W_.a___))
                .with(Atom::num(1));
        });
    }
    exprs.push(ParametricIntegrands { integrands, cuts });
}

Output Assembly

Hedge-poset fetches stored final integrands by operation and sums them into the cut result.

for nidx in self.compatible_topological_order(cut_subset)? {
    let operation = &self.graph[nidx];
    let computed = self.compute_store.require(operation)?;
    let Integrands::Multiple(final_integrand) =
        &computed.final_integrand
    else { ... };

    let sigma = settings.add_sigma
        .then(|| function!(GS.if_sigma, operation.to_atom()));
    let sum = sum.get_or_insert_with(ResidueSelectedTerms::empty);

    for (cut_index, integrand) in final_integrand {
        let mut term = integrand.clone();
        if let Some(sigma) = &sigma {
            term *= sigma;
        }
        sum.add_assign_term(*cut_index, term);
    }
}

Shared Final-Integrand Construction

Both orchestrators use the same final-integrand builder through ForestNodeLike and FinalIntegrandNode. This is the reuse boundary: it depends on the forest-node invariant, not on the legacy Approximation storage layout. 

pub(crate) trait FinalIntegrandNode: ForestNodeLike {
    fn final_integrand_builder<'a>(
        &'a self,
        local_3d: &'a ResidueSelectedTerms,
        cutset: &'a CutSet,
        valid_orientations: &'a [EdgeVec<Orientation>],
        orientation_pattern: &'a OrientationPattern,
        debug_label: Atom,
    ) -> FinalIntegrandBuilder<'a, Self>
    where
        Self: Sized;
}

Current Status

The hedge-poset path has the structure needed for component-wise local traversal: operation levels, a root/frontier-aware compatible order, and a shared final-integrand construction path. Integrated disconnected spinneys should factorize as products of component integrated terms, but the full spectacles regression is not validated yet and currently exposes Vakint topology/momentum-mapping issues. Local 3D disconnected composition is not implemented semantically: it must be made operational so each disconnected component acts from the common ancestor/root without duplicating that ancestor in the product. Dedicated integration tests should cover spectacles integrated factorization and local 3D common-root composition separately.
